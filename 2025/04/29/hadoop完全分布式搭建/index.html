hadoop完全分布式环境搭建

搭建一个包含 NameNode、SecondaryNameNode 和 ResourceManager 分布在三台机器（wpx001, wpx002, wpx003）上的 Hadoop 分布式平台，通常涉及 HDFS 和 YARN 两个核心组件。

下面是详细步骤：

规划节点角色:

  • wpx001:
    • NameNode (HDFS Master)
    • ResourceManager (YARN Master)
    • DataNode (HDFS Worker)
    • NodeManager (YARN Worker)
  • wpx002:
    • SecondaryNameNode (HDFS Checkpointer)
    • DataNode (HDFS Worker)
    • NodeManager (YARN Worker)
  • wpx003:
    • DataNode (HDFS Worker)
    • NodeManager (YARN Worker)

前提条件 (在所有三台机器上执行):

  1. 操作系统: 确保所有节点使用兼容的 Linux 发行版（如 CentOS, Ubuntu）。

  2. 网络配置:

    • 配置静态 IP 地址（推荐）。

    • 确保所有节点主机名设置正确 (hostnamectl set-hostname wpx00X)。

    • 配置 /etc/hosts 文件，使所有节点都能通过主机名互相解析。文件内容类似：

    • 确保节点之间网络互通（ping wpx001, ping wpx002, ping wpx003 都能成功）。

    • 关闭或配置防火墙，允许 Hadoop 所需端口通信（常见端口：9000, 9870, 9868, 8088, 8030-8033, 8040, 8042 等，具体取决于 Hadoop 版本和配置）。简单起见，可以先禁用防火墙：sudo systemctl stop firewalld && sudo systemctl disable firewalld (CentOS/RHEL) 或 sudo ufw disable (Ubuntu)。生产环境请精细配置规则！

  3. 创建 Hadoop 用户: 创建一个专门的用户来运行 Hadoop 进程（例如 hadoop）。

    确保三台机器上的 hadoop 用户具有相同的 UID 和 GID (可以使用 id hadoop 查看并用 usermod -u <UID>, groupmod -g <GID> 调整)。

  4. 安装 Java: 安装合适的 JDK（Hadoop 3.x 通常需要 Java 8 或 11）。确保 JAVA_HOME 环境变量已设置，并且对 hadoop 用户生效（可以在 /etc/profile 或 ~/.bashrc 中设置）。

  5. 配置 SSH 免密登录: Hadoop 的启动/停止脚本需要从主节点（这里是 wpx001）免密登录到所有其他节点（包括自身）。

    • 在 wpx001 上，切换到 hadoop 用户：su - hadoop

    • 生成 SSH 密钥（如果还没有）：ssh-keygen -t rsa (一路回车)

    • 将公钥复制到所有节点（包括 wpx001 自身）：

      （过程中需要输入各节点 hadoop 用户的密码）

    • 测试免密登录：ssh wpx001, ssh wpx002, ssh wpx003，应该都不需要密码直接登录。

验证集群状态:

  1. 使用 jps 命令检查进程:

    • 在 wpx001 上运行 jps，应看到：NameNode, ResourceManager, DataNode, NodeManager (可能还有 Jps 本身)。
    • 在 wpx002 上运行 jps，应看到：SecondaryNameNode, DataNode, NodeManager, Jps。
    • 在 wpx003 上运行 jps，应看到：DataNode, NodeManager, Jps。

  2. 访问 Web UI: 在浏览器中打开以下地址：

    • YARN ResourceManager UI: http://wpx001:8088。查看 Nodes 状态，应该有 3 个 Active Nodes。

    • HDFS SecondaryNameNode UI: http://wpx002:9868 (Hadoop 3.x 默认端口，旧版可能是 50090)。

常见问题和注意事项:

  • 日志文件: 如果启动失败或运行异常，检查 Hadoop 日志文件，通常位于 $HADOOP_HOME/logs 目录下。
  • 权限问题: 确保 hadoop 用户对 Hadoop 安装目录、数据目录、临时目录有读写权限。
  • 端口冲突: 确保 Hadoop 需要的端口没有被其他服务占用。
  • 内存配置: 对于实际应用，你可能需要调整 yarn-site.xml 和 mapred-site.xml 中的内存相关配置（如 yarn.nodemanager.resource.memory-mb, yarn.scheduler.maximum-allocation-mb, mapreduce.map.memory.mb, mapreduce.reduce.memory.mb 等）。
  • 一致性: 确保所有节点的 Java 版本、Hadoop 版本、配置文件（除了特定于节点的目录）都是一致的。
